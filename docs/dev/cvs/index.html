cvs.perl.org is the CVS repository for a number of Perl projects, including [% module %]. [% IF module.match("^(parrot|mod_parrot|perlfaq)$") %]

Mailinglist

All commits gets mailed to the cvs-[% module %] mailinglist. Be sure to subscribe if you are interested in what's going on. [% END %]

Anonymous CVS access


  cvs -d :pserver:anonymous@cvs.perl.org:/cvs/public login
  [press enter at the password prompt]
  cvs -d :pserver:anonymous@cvs.perl.org:/cvs/public co [% module %]

Web interface to the CVS repository

ViewCVS is available at http://cvs.perl.org/viewcvs/[% module %].

For registered developers

First, if you don't already have it get a perl.org account. The maintainers of [% module %] then need to tell cvs@perl.org to enable CVS access for your account.

After you have had CVS access enabled on your account, please allow up to 30 minutes before all systems are synchronized (likewise if you later change your password).

Check out a local copy of the [% module %] module

Where LOGIN is your perl.org login name, do a 
   cvs -d :pserver:LOGIN@cvs.perl.org:/cvs/public login
   [enter your perl.org password]
   cvs -d :pserver:LOGIN@cvs.perl.org:/cvs/public co [% module %]
   cd [% module %]

Keep your copy updated

  cvs update -dP

Commit some changes

  cvs commit filename [filename]

Do a diff

  cvs diff [filename]

Add a file

  cvs add [filename]

Other tips

I like the "-u -d" options to diff, and to use gzip compression when updating, so I have 
 diff -u -d
 update -z3
in my ~/.cvsrc.

Snapshots

If you cannot access the CVS server, for whatever reason, firewall or just company policy, you can access CVS snapshots, created every 6 hours.

http://cvs.perl.org/snapshots/
