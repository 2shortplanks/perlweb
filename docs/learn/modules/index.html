Perl modules, what are they and why do I care?

What is a module?

A Perl module is a set of related functions in a library file, i.e., a Perl package with the same name as the file. It is specifically designed to be reusable by other modules or programs. CPAN has [% perl_modules | comma %] modules ready for you to use in just this way.

Most Perl modules are written in Perl, some use XS (they are written in C) so require a C compiler (if you followed the installing instructions you already have one). Modules may have dependencies on other modules (almost always on CPAN) and cannot be installed without them (or without a specific version of them). Many modules on CPAN now require a recent version of Perl (version 5.8 or above).

Where can I find modules?

https://metacpan.org/ lets you search the [% perl_stats.cpan_modules | comma %] modules on CPAN.

Which module should I use?

Metacpan has user ratings, also have a look at Task::Kensho. The examples section and FAQ also have some recommendations, you could also ask the Perl community.

How do I install a module?

If you haven't already install cpanm do that first:

[% WRAPPER code_snippet %]cpan App::cpanminus[% END %]

Then to install any module from CPAN

[% WRAPPER code_snippet %]cpanm Module::Name[% END %]
