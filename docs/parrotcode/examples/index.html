Introduction

Parrot is a virtual machine used to efficiently execute bytecode for interpreted languages - specifically, Perl 6, although we would like to keep the door open for other languages in the future. As such, Parrot will be the core of the Perl interpreter; it will be the target platform to which Perl 6 code is compiled.

Parrot currently consists of an assembler and a virtual machine. The best introduction to the Parrot virtual machine is an article by Simon Cozens, Parrot: Some Assembly Required, and it is recommended that you read that before continuing. Note that the pace of Parrot development is currently quite fast and some of the details in the article are out of date. This guide will attempt to keep up to date to the latest CVS Parrot sources.

Parrot is intended to be the virtual machine for Perl 6, but it is not tied to Perl 6. In fact, we hope Parrot can be a valid target architecture and common language runtime for many languages. This can only be achieved through detailed Parrot examples and explanations - the purpose of this site.

Download the latest version of Parrot if you have not already done so.

Since Parrot provides two languages to generate bytecode, there are two separate sets of examples:

Parrot assembler

Parrot assembler (PASM) is a low-level language which compiles directly into the Parrot virtual machine. Unlike most other virtual machines (but like most real computers), Parrot is register-based. It has 32 registers of four different types: integers, numbers, strings, and PMCs. These are named I0...I31, N0...N31, S0...S31, and P0...P31. Most Parrot operators have the same format: the name of the operator, the destination register, and the operands.

The Parrot assembly language documentation (PDD6) is the main source of information about the assembly language, so you should read that for full information about the language.

Parrot Intermediate Language

Parrot Intermediate Language (abbreviated PIR) (No, really) is a slightly higher level language than PASM. It manages the Parrot Calling Conventions (PDD03) transparently, provides improved syntax, register allocation, register spilling, and more.
