Parrot 0.1.0 "Leaping Kakapo" Released!

On 29 February 2004, Parrot pumpking Leo Tötsch released Parrot 0.1.0. See the Parrot 0.10 release announcement for more details or download Parrot 0.1.0. from the CPAN

What is Parrot?

Parrot is a virtual machine designed to execute bytecode for interpreted languages efficiently. Parrot will be the target for the Perl 6 compiler. There is already a partial Perl 6 compiler as well as compilers in various stages of completion for a wide range of other languages.

Where Are We?

The Parrot source code was first released to the world on Monday 10th of September 2001. Since then, an army of developers has been furiously hacking away on it. Currently you can compile Parrot assembly and IMC (an intermediate language) to Parrot bytecode and execute it. Objects, threads, garbage collection, a JIT compiler, namespaces, subroutines, a range of basic types (e.g., arrays and hashes) and NCI (for calling into native libraries) are all available.

Sean O'Rourke has contributed a working Perl 6 parser and compiler for a significant subset of the Perl 6 definition that currently exists. The languages directory of the distribution contains other compilers for you to look at.

To keep up to date on the latest developments, check out This week on Perl 6 by Piers Cawley, which covers the Parrot and Perl 6 mailing lists.

How to get the code

Periodic releases will appear on CPAN; the current release is Parrot version 0.1.0.

For the very latest version of Parrot, check out the source from CVS. There is anonymous access to all, and commit access to those who have taken responsibility for various areas of the code or produced a large number of high-quality patches.

Frequent automatic Parrot snapshots are also available.

For automatic smoke testing you can use rsync to synchronize a directory with a checked-out copy of the latest CVS tree.

rsync -av --delete cvs.perl.org::parrot-HEAD parrot

How to Get Involved

Parrot development takes place on the perl6-internals mailing list. If you follow this list, you'll be able to get an idea of what sort of things are needed. Subscribe by sending mail to perl6-internals-subscribe@perl.org or read the perl6-internals NNTP archive.

There is documentation in the docs subdirectory of the Parrot source. You can also read the Parrot design documents.

Patches of all natures are welcome. See the patch FAQ for details.

View and manipulate bugs and to-do items at the Parrot bugs page.
