A good setup for Perl on Unix/Linux based systems is as follows.

  • Install a compiler, such as gcc through your system package management (e.g. XXX, YYY).
  • Open a Terminal and copy & paste the following command below into it (then press the 'return' key).
  • [% WRAPPER code_snippet %]curl -L http://xrl.us/installperlnix | bash[% END %]

Explanation of this setup

  • To build and install Perl and many of the thousands of useful Perl modules you need to have a compiler.
  • To interact with the command line and run Perl commands, a terminal application is needed. OSX comes with Terminal.app by default, there are alternatives available as well.
  • The Perlbrew website lists many of it's advantages. We recommend it here because it separates your installation from the system Perl, this makes upgrading your OS less likely to cause issues with your own Perl setup. It also means you can use a newer Perl than the default one which comes with your OS.
  • Perlbrew does let you have multiple versions of Perl installed (very useful for testing if you want to upgrade to a new version), for the purposes of learning Perl you will only need one version which is why we 'switch' to use this as the default.
  • App-cpanminus provides the cpanm tool, which makes installing modules very simple to do.
