A good setup for Perl on OSX is as follows.

  • Install Xcode through the Mac App Store (Apple charge a small amount for this, it is a very large download), or from your OSX software DVD/SSD if possible (any version will work).
  • Open the Terminal.app (Found in Applications -> Utilities) and copy & paste the following command below into it (then press the 'return' key).
  • [% WRAPPER code_snippet %]curl -L http://xrl.us/installperlosx | bash[% END %]

Explanation of this setup

  • To build and install Perl and many of the thousands of useful Perl modules you need to have a compiler, for OSX the easiest way to get this is to use Apple's Xcode. Some versions of the OSX install DVDs or SSDs come with the 'developer tools', you can install using these, only 'unix tools' is actually required. No specific version of Xcode is needed, the latest is available directly through the App Store.
  • To interact with the command line and run Perl commands, a terminal application is needed. OSX comes with Terminal.app by default, there are alternatives available as well.
  • OSX uses .bash_profile, but most other systems (and instructions) look for .bashrc, this code sets up .bash_profie to also run anything in .bashrc. There are other differences but these are not important to us.
  • The Perlbrew website lists many of it's advantages. We recommend it here because it separates your installation from the system Perl, this makes upgrading OSX less likely to cause issues with your own Perl setup. It also means you can use a newer Perl than the default one which comes with OSX.
  • Perlbrew does let you have multiple versions of Perl installed (very useful for testing if you want to upgrade to a new version), for the purposes of learning Perl you will only need one version which is why we 'switch' to use this as the default.
  • App-cpanminus provides the cpanm tool, which makes installing modules very simple to do.
